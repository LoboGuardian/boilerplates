# cli-ts

Node.js CLI tool + TypeScript + Commander

## Idea

A starter for building distributable CLI tools with Node.js. Publishable to npm, with a clean command structure, colored output, and proper error handling.

## Stack

- **TypeScript** — compiled to CJS or ESM
- **Commander.js** — command and option parsing
- **Inquirer** — interactive prompts
- **Chalk** — terminal color output
- **Ora** — spinner for async operations
- **Zod** — config file validation
- **tsup** — bundler (zero-config, outputs CJS + ESM)
- **Vitest** — testing
- **ESLint + Prettier** — linting and formatting

## Structure

```
src/
  commands/       # One file per CLI command
    init.ts
    build.ts
  lib/            # Shared utilities
    logger.ts     # Chalk-based logger (info, warn, error)
    config.ts     # Config file loading and validation
    fs.ts         # File system helpers
  types/          # Shared TypeScript types
  index.ts        # Entry point — registers all commands
bin/
  cli.js          # Shebang wrapper (generated by tsup)
```

## Decisions

- `tsup` handles the build: single command outputs to `dist/`, sets the shebang, and marks the package as executable.
- `package.json` `bin` field points to `dist/index.js`.
- Commands are registered in `index.ts` — adding a command is a single import and `.addCommand()` call.
- Config files (`.mytoolrc.json`) are validated with Zod so users get clear error messages for invalid config.
- Errors are caught at the top level and displayed with `chalk.red` — never raw stack traces to the user.
